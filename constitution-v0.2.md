# AI Organization Constitution v0.2

## Status
Draft.

## Inheritance note
This version inherits and compresses hard requirements from `aion-governance` into a three-role AI-only operating model.

## Foundational roles
- Monarch
- Operator Core
- Audit Core

## Constitutional laws
1. GitHub is the formal governance center.
2. Discord is the short command, routing, and state-return layer.
3. The Monarch is not a manual relay layer.
4. Documents are AI-first and human-light by default.
5. Minimal reading packs outrank full-library replay.
6. Execution is not acceptance.
7. Audit is not implementation.
8. High-risk work requires stop conditions, rollback or recovery thinking, and formal write-back.
9. Every non-trivial task must have explicit owner, boundary, and write-back target.
10. Silence is a diagnosable system state, not an acceptable ambiguity.
11. Token waste is a governance defect.
12. Rules may evolve, but only through formal recorded change.

## Human-light rule
The organization should minimize routine human relay, monitoring, and content transport. Human involvement should concentrate on direction, constitutional approval, and exceptional intervention.

## Active phase rule
In the current phase, the three-role world is the official minimum operating structure. Future role splits are allowed only when complexity justifies them.
