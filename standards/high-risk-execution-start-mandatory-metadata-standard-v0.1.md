# High-Risk Execution Start Mandatory Metadata Standard v0.1

## Purpose
Define the minimum metadata that must exist before any high-risk issue enters execution.

## Mandatory fields before execution
Every high-risk issue must declare all of the following before execution begins:
1. risk category
2. evidence matrix row
3. primary owner
4. write-back target
5. rollback note or dependent-risk note where applicable

## Execution start rule
If any mandatory field is missing, the issue may be shaped or discussed, but it must not enter execution state.

## Why this exists
This rule is designed to hard-freeze proof obligations before work starts, reducing hidden self-certification and closure drift.
