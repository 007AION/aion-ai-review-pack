# Token Economy & Minimal Reading Pack Standard v0.1

## Purpose
Prevent governance growth from forcing massive repetitive reading.

## Core principle
Do not require agents to read the whole library to perform a bounded task.

## Reading hierarchy
### Tier 1: Always-small
Read only:
- current issue
- minimal reading pack
- immediate evidence target

### Tier 2: Conditional
Read only if needed:
- one related standard
- one related previous issue or PR
- one runtime note

### Tier 3: Exceptional
Read wider governance only when:
- constitutional conflict exists
- role boundary is unclear
- audit requests wider review

## Authoring rule
Every standard should be:
- short
- scoped
- routable
- linkable
- not dependent on full-library replay

## Compression rule
When a pattern is stable, convert long narrative into:
- checklist
- template
- taxonomy
- decision table

## Forbidden anti-patterns
- giant master docs required for all tasks
- repeated full-governance reading
- closure that depends on historical memory not formalized in records
