# Historical Governance Migration Standard v0.1

## Purpose
Define how historical governance requirements from `aion-governance` are transferred into the active three-role AI-only prototype.

## Classification tags
Each inherited rule should be tagged as one of:
- **preserve**: keep with no weakening
- **compress**: keep intent, reduce role or document complexity
- **delay**: keep for later maturity stage
- **retire**: not useful for the present phase

## Current migration decisions

### Preserve
1. GitHub as formal governance center
2. Discord as short routing layer
3. no manual relay burden on Monarch
4. minimal reading pack discipline
5. execution / audit / acceptance separation
6. rollback, stop condition, write-back discipline
7. explicit dispatch and handoff
8. silence / stall diagnosis
9. token economy as governance rule

### Compress
1. 8-role surface -> 3-role world
2. General manager / chief of staff / intelligence / knowledge -> Operator Core subfunctions
3. Runtime lead / field agent -> Operator execution modes
4. detailed hierarchy language -> lighter workflow-oriented standards

### Delay
1. full role-fit reassignment framework
2. rich fallback hierarchy details beyond current minimum
3. broader specialization patterns for larger future departments

### Retire
1. any historical wording that adds ceremony without increasing clarity or safety
2. any structural complexity that forces unnecessary human coordination in the current phase

## Migration law
A compressed rule must preserve the original control objective, even if wording and structure change.
